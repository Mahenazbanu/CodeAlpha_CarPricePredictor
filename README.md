# Car Price Predictor

A machine learning application that estimates used vehicle resale prices in the Indian automobile market. Given vehicle attributes, it trains four regression models, automatically selects the best performer by R² score, and serves predictions through a Gradio web interface.

---

├── gui.py                # Gradio web interface

## Dataset

**File:** `car_data.csv`  
**Records:** 301 vehicles (cars and motorcycles)  
**Market:** Indian automobile market

| Column | Type | Description |
|---|---|---|
| `Car_Name` | String | Vehicle model name |
| `Year` | Integer | Manufacturing year |
| `Present_Price` | Float | Current showroom price (₹ Lakhs) |
| `Selling_Price` | Float | Resale price — **prediction target** (₹ Lakhs) |
| `Driven_kms` | Integer | Total distance driven (km) — renamed to `Kms_Driven` during preprocessing |
| `Fuel_Type` | Categorical | Petrol / Diesel / CNG |
| `Selling_type` | Categorical | Dealer / Individual — renamed to `Seller_Type` during preprocessing |
| `Transmission` | Categorical | Manual / Automatic |
| `Owner` | Integer | Number of previous owners |

## Usage

**Step 1 — Train the model**

```bash
python train_model.py
```

Reads `car_data.csv`, trains all four models, prints a comparison table, and saves `car_price_model.pkl`.

Optional arguments:

```bash
python train_model.py --data <path/to/csv> --out <path/to/output.pkl>
```

**Step 2 — Launch the GUI**

```bash
python gui.py
```

Opens a browser window at `http://127.0.0.1:7860`. On Android (Pydroid 3), navigate to that address manually.

> `car_price_model.pkl` must exist before launching `gui.py`. Run `train_model.py` first if the file is missing.

## How It Works

### Training (`train_model.py`)

1. Loads `car_data.csv` and strips whitespace from column names.
2. Renames `Selling_type` → `Seller_Type` and `Driven_kms` → `Kms_Driven` for consistency.
3. Engineers a `Car_Age` feature: `Car_Age = 2026 − Year`.
4. Label-encodes four categorical columns: `Car_Name`, `Fuel_Type`, `Seller_Type`, `Transmission`.
5. Selects eight features for training:
   `Car_Name_enc`, `Car_Age`, `Present_Price`, `Kms_Driven`, `Fuel_Type_enc`, `Seller_Type_enc`, `Transmission_enc`, `Owner`
6. Splits data 80/20 (`random_state=42`).
7. Trains all four models and evaluates each on the test set (R², MAE, RMSE).
8. Selects the model with the highest R² score as the best model.
9. Saves a bundle to `car_price_model.pkl` containing:
   - `model` — the best-performing sklearn estimator
   - `best_name` — name of the best model
   - `encoders` — dict of four `LabelEncoder` objects (`fuel`, `seller`, `transmission`, `car_name`)
   - `feature_names` — ordered list of input feature names
   - `results` — metrics (R², MAE, RMSE, y_test, y_pred) for all four models
   - `current_year` — `2026`
   - `car_names` — sorted list of all vehicle names for the GUI dropdown

### GUI (`gui.py`)

The Gradio interface has two tabs:

**Predict Price tab**
- Dropdown for car name (populated from `bundle["car_names"]`)
- Slider for manufacturing year (1995–2026)
- Number inputs for current showroom price and kilometres driven
- Slider for previous owners (0–3)
- Dropdowns for fuel type, seller type, and transmission (options read from encoders)
- On submit: applies the same label encoding and car age calculation, constructs a feature array in training order, calls `model.predict()`, and displays a Plotly gauge chart plus an HTML result card

**Accuracy & Insights tab** (loaded automatically on startup)
- Bar chart comparing R², MAE, and RMSE across all four models
- Actual vs predicted scatter plot for the best model
- Residual distribution histogram for the best model
- Ranked metrics table with the best model highlighted

## Model Information

Four regression algorithms are trained and compared on each run:

| Algorithm | scikit-learn class |
|---|---|
| Linear Regression | `LinearRegression()` |
| Decision Tree | `DecisionTreeRegressor(random_state=42)` |
| Random Forest | `RandomForestRegressor(n_estimators=100, random_state=42)` |
| Gradient Boosting | `GradientBoostingRegressor(n_estimators=100, random_state=42)` |

The model with the highest R² on the 20% test set is saved as the production model. Exact metric values vary per run; the GUI's Accuracy tab always reflects the metrics from the current trained model.

**Evaluation metrics:** R² (coefficient of determination), MAE (mean absolute error, ₹ Lakhs), RMSE (root mean squared error, ₹ Lakhs).

## Troubleshooting

**`❌ Model file not found`** — Run `python train_model.py` before `python gui.py`. Both files must be in the same directory.

**`ModuleNotFoundError`** — Install the missing package: `pip install -r requirements.txt`

**`⚠️ Car 'X' was not seen during training`** — Only vehicle names present in `car_data.csv` are valid. Use the dropdown to select a known model.

**Android / Pydroid 3** — If the browser does not open automatically, navigate to `http://127.0.0.1:7860` or `http://localhost:7860` manually.
